msiempy API Documentation

Class Hierarchy

  • abc.ABC
    • msiempy.core.types.NitroObject - Base class for all nitro objects. All objects have a reference to the single msiempy.core.session.NitroSession object that handle the esm requests. Creates the object session.
      • msiempy.core.types.NitroDict - Dict-Like object (Base class) to represent SIEM data. Exemple : Event, Alarm, etc...
        • msiempy.alarm.Alarm - Dict-Like object. Represents a triggered alarm.
        • msiempy.device.DataSource - Dict-Like object.
        • msiempy.event.Event - Dict-Like object. Represents an event in the SIEM.
          • msiempy.event.GroupedEvent - Dict-Like object. Represents a row of grouped query results.
        • msiempy.watchlist.Watchlist - Dict-Like object. Represent a ESM Watchlist.
      • msiempy.core.types.NitroList - Base class for list objects.
        • msiempy.core.query.FilteredQueryList - Abstract base class that provide time range handling and define the query filters.
          • msiempy.alarm.AlarmManager - List-Like object. Interface to query and manage alarms.
          • msiempy.event._QueryExecuteManager - Abstract class to handle common filters properties that grouped and non-grouped queries share.
            • msiempy.event.EventManager - List-Like object. Interface to execute a event query.
            • msiempy.event.GroupedEventManager - List-Like object. Interface to execute a grouped event query.
        • msiempy.device.DevTree - List-Like object. ESM device tree interface.
        • msiempy.watchlist.WatchlistManager - List-Like object. Summary of ESM watchlists.
      • msiempy.device.ESM - Enterprise Security Manager interface.
  • collections.UserDict
    • msiempy.core.types.NitroDict - Dict-Like object (Base class) to represent SIEM data. Exemple : Event, Alarm, etc...
      • msiempy.alarm.Alarm - Dict-Like object. Represents a triggered alarm.
      • msiempy.device.DataSource - Dict-Like object.
      • msiempy.event.Event - Dict-Like object. Represents an event in the SIEM.
        • msiempy.event.GroupedEvent - Dict-Like object. Represents a row of grouped query results.
      • msiempy.watchlist.Watchlist - Dict-Like object. Represent a ESM Watchlist.
    • msiempy.event._QueryFilter - Base class for all SIEM query objects in order to dump the filter as dict.
      • msiempy.event.FieldFilter - Query field filter
      • msiempy.event.GroupFilter - Query group filter
  • collections.UserList
    • msiempy.core.types.NitroList - Base class for list objects.
      • msiempy.core.query.FilteredQueryList - Abstract base class that provide time range handling and define the query filters.
        • msiempy.alarm.AlarmManager - List-Like object. Interface to query and manage alarms.
        • msiempy.event._QueryExecuteManager - Abstract class to handle common filters properties that grouped and non-grouped queries share.
          • msiempy.event.EventManager - List-Like object. Interface to execute a event query.
          • msiempy.event.GroupedEventManager - List-Like object. Interface to execute a grouped event query.
      • msiempy.device.DevTree - List-Like object. ESM device tree interface.
      • msiempy.watchlist.WatchlistManager - List-Like object. Summary of ESM watchlists.
  • configparser.ConfigParser
    • msiempy.core.config.NitroConfig - configparser.ConfigParser parser object.
  • Exception
    • msiempy.core.session.NitroError - Base ESM exception. It's used when the user/passwd is incorrect and other HTTP errors.
  • json.JSONEncoder
    • msiempy.core.types.NitroObject.NitroJSONEncoder - Custom JSON encoder that will use the approprtiate propertie depending of the type of NitroObject. TODO support json json dumping of QueryFilers, may be by making them inherits from NitroDict.
  • msiempy.core.session.NitroSession - Single session handler and HTTP interface.

msiempy API Documentation

Class Hierarchy

  • abc.ABC
    • msiempy.core.types.NitroObject - Base class for all nitro objects. All objects have a reference to the single msiempy.core.session.NitroSession object that handle the esm requests. Creates the object session.
      • msiempy.core.types.NitroDict - Dict-Like object (Base class) to represent SIEM data. Exemple : Event, Alarm, etc...
        • msiempy.alarm.Alarm - Dict-Like object. Represents a triggered alarm.
        • msiempy.device.DataSource - Dict-Like object.
        • msiempy.event.Event - Dict-Like object.
          • msiempy.event.GroupedEvent - Dict-Like object. Grouped event item, represents a row of grouped query results.
        • msiempy.watchlist.Watchlist - Dict-Like object.
      • msiempy.core.types.NitroList - Base class for list objects.
        • msiempy.core.query.FilteredQueryList - Base class for query based managers : msiempy.alarm.AlarmManager, msiempy.event.EventManager. FilteredQueryList object can handle time_ranges and time splitting. Abstract base class that provide time ranged filtered query wrapper.
          • msiempy.alarm.AlarmManager - Interface to query and manage Alarms. Inherits from msiempy.core.query.FilteredQueryList.
          • msiempy.event._QueryExecuteManager - Abstract class to handle common filters properties that grouped and non-grouped queries share. As well as providing helper functions to wait the query and get the results (see source code). Only Events query are currently supported.
            • msiempy.event.EventManager - List-Like object.
            • msiempy.event.GroupedEventManager - List-Like object. Interface to execute a grouped event query.
        • msiempy.device.DevTree - List-Like object. ESM device tree interface.
        • msiempy.watchlist.WatchlistManager - List-Like object.
      • msiempy.device.ESM - Enterprise Security Manager interface. Object do not contain data, it's a simple interface to data structures / values returned by the SIEM or helper methods.
  • collections.UserDict
    • msiempy.core.types.NitroDict - Dict-Like object (Base class) to represent SIEM data. Exemple : Event, Alarm, etc...
      • msiempy.alarm.Alarm - Dict-Like object. Represents a triggered alarm.
      • msiempy.device.DataSource - Dict-Like object.
      • msiempy.event.Event - Dict-Like object.
        • msiempy.event.GroupedEvent - Dict-Like object. Grouped event item, represents a row of grouped query results.
      • msiempy.watchlist.Watchlist - Dict-Like object.
    • msiempy.event._QueryFilter - Base class for all SIEM query objects in order to dump the filter as dict.
      • msiempy.event.FieldFilter - Based on EsmFieldFilter SIEM api doc.
      • msiempy.event.GroupFilter - Based on EsmFilterGroup. See SIEM api doc. Used to dump groups of filters in the right format.
  • collections.UserList
    • msiempy.core.types.NitroList - Base class for list objects.
      • msiempy.core.query.FilteredQueryList - Base class for query based managers : msiempy.alarm.AlarmManager, msiempy.event.EventManager. FilteredQueryList object can handle time_ranges and time splitting. Abstract base class that provide time ranged filtered query wrapper.
        • msiempy.alarm.AlarmManager - Interface to query and manage Alarms. Inherits from msiempy.core.query.FilteredQueryList.
        • msiempy.event._QueryExecuteManager - Abstract class to handle common filters properties that grouped and non-grouped queries share. As well as providing helper functions to wait the query and get the results (see source code). Only Events query are currently supported.
          • msiempy.event.EventManager - List-Like object.
          • msiempy.event.GroupedEventManager - List-Like object. Interface to execute a grouped event query.
      • msiempy.device.DevTree - List-Like object. ESM device tree interface.
      • msiempy.watchlist.WatchlistManager - List-Like object.
  • configparser.ConfigParser
    • msiempy.core.config.NitroConfig - Handles the configuration. Reads the config file .msiem/conf.ini where ever it is and make accessible it's values throught object properties. If a .msiem/ directory exists in your current directory, the program will assume the conf.ini file is there, if not, it will create it with default values. Secondly, if no .msiem/ directory exists in the current directory, it will be automatically placed in a appropriate place depending of your platform:
  • Exception
    • msiempy.core.session.NitroError - Base ESM exception. It's used when the user/passwd is incorrect and other HTTP errors.
  • json.JSONEncoder
    • msiempy.core.types.NitroObject.NitroJSONEncoder - Custom JSON encoder that will use the approprtiate propertie depending of the type of NitroObject. TODO support json json dumping of QueryFilers, may be by making them inherits from NitroDict.
  • msiempy.core.session.NitroSession - msiempy.core.session.NitroSession is the point of convergence of every requests that goes to the ESM. It provides easier dialogue with the ESM by doing argument interpolation with msiempy.core.session.NitroSession.PARAMS.
